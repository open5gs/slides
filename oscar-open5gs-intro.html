Our Humble Beginnings

  • Feb 1, 2017: Started coding from scratch
  • Pure hobby—no external support
  • 10 months until first release
  • Collaborated with Jihoon Lee & Jinyoung Park
  • Forked as Open5GS; 5G stack launched in 2020
On February 1, 2017, I started coding Open5GS from scratch as a pure hobby project. After 10 months of work with help from friends Jihoon Lee and Jinyoung Park, we released the first version. While they later founded NextEPC for commercial purposes, I forked the project as Open5GS to maintain the open source spirit, adding our 5G stack in 2020.

Overview of Open5GS Functionality

  • Supports both 4G EPC and 5G Core Networks
  • Modular, service-based design
  • Key components: AMF, SMF, UPF, NRF, SEPP, etc.
  • Robust protocol support (HTTP/2, TLS, PFCP, Diameter)
Open5GS bridges 4G and 5G technologies with its modular, service-based architecture. You can deploy, upgrade, and integrate each component independently—perfect for legacy 4G systems while preparing for 5G. The core includes AMF for access management, SMF for session management, and UPF for user plane functions, all supported by robust protocols.

Architecture & Core Components

Flexible, Modular, and Secure

  • Common libraries: core, security, s1ap, ngap, nas, etc.
  • Dedicated directories for network functions: amf, mme, smf, upf, etc.
  • Seamless integration with legacy 4G systems
  • Advanced security with TLS and encrypted identifiers (e.g., SUCI)
Open5GS is built with clarity and reusability in mind. We maintain common libraries like core, security, s1ap, ngap, and nas as the foundation. Each network function is organized into dedicated directories, making development easy. Our architecture supports seamless 4G integration while providing advanced security with TLS and encrypted identifiers like SUCI.
